# 제주도 여름철 전력 수요 예측

> 폭염 및 열대야 심화에 따른 제주도 여름철 전력 수요 변동을 예측하기 위한 머신러닝 기반 회귀 모델링 프로젝트입니다.  
> 본 프로젝트는 **전력 피크 시점 탐지**가 아니라, 시간별 **전력 수요량(MWh)** 자체를 예측하는 것을 목표로 합니다.

## 1. 프로젝트 개요

최근 기후변화로 여름철 폭염과 열대야가 심화되면서 냉방 수요가 증가하고 있습니다. 특히 제주도는 섬 지역이라는 전력 공급 특성과 관광객 유입에 따른 수요 변동성이 함께 존재하기 때문에, 안정적인 전력 수급 관리를 위해 시간별 전력 수요 예측이 중요합니다.

본 프로젝트에서는 제주도 여름철 시간별 전력 수요 데이터를 중심으로 기상, 인구, 관광객, 재생에너지 발전량, 과거 전력 수요 파생변수를 결합하여 전력 수요 예측 모델을 구축했습니다.

## 2. 데이터 출처 및 구성

본 프로젝트는 제주도 여름철 시간별 전력 수요 예측을 위해 전력 수요, 기상, 인구, 관광객, 재생에너지 발전량 데이터를 수집하여 활용했습니다. 각 데이터는 날짜 및 시간 기준으로 병합했으며, 결측치 처리와 파생변수 생성을 거쳐 최종 모델링 데이터셋을 구성했습니다.

| 데이터 구분 | 주요 변수 | 출처 |
|---|---|---|
| 전력 수요 데이터 | 시간별 전력 수요량(`power_demand_MWh`) | 한국전력거래소 / 공공데이터포털 |
| 기상 데이터 | 평균기온, 최고기온, 최저기온, 평균습도, 최고습도, 최저습도 등 | 기상청 기상자료개방포털 |
| 인구 데이터 | 거주인구, 근무인구, 방문인구 | 공공데이터포털 / 제주데이터허브 |
| 관광객 데이터 | 내국인 관광객, 외국인 관광객, 총 관광객 수 | 제주관광공사 / 제주특별자치도 관광 통계 |
| 재생에너지 데이터 | 태양광 발전량, 풍력 발전량, 총 재생에너지 발전량 | 한국전력거래소 / 공공데이터포털 |

원본 데이터는 각 기관에서 제공하는 공개 데이터를 기반으로 하며, 본 저장소에는 분석 코드와 모델링 과정을 중심으로 정리했습니다. 

## 3. 분석 목표

- 제주도 여름철 시간별 전력 수요량(`power_demand_MWh`) 예측
- 단순 과거 전력 수요 변수만 사용한 모델과 다변량 모델의 성능 비교
- Linear Regression과 XGBoost Regressor를 활용한 회귀 모델링
- 변수 중요도 및 회귀계수 해석을 통한 전력 수요 영향 요인 도출

## 4. Train/Test 분할

본 프로젝트의 데이터는 시간 순서가 중요한 시계열 성격을 가지므로, 랜덤 분할을 사용하지 않고 **시간 순서를 유지한 채 Train/Test를 분할**했습니다. 모델이 미래 데이터를 미리 학습하는 데이터 누수를 방지하기 위해, 마지막 연도인 2025년 데이터를 테스트 데이터로 사용했습니다.

| 구분 | 기간 | 행 수 | 설명 |
|---|---|---:|---|
| Train 데이터 | 2020~2024년 7~8월 | 7,440행 | 모델 학습에 사용 |
| Test 데이터 | 2025년 7~8월 | 1,488행 | 최종 성능 평가에 사용 |

코드에서는 2025년 7~8월을 `62일 × 24시간 = 1,488행`으로 계산하고, 전체 데이터의 마지막 1,488행을 test 데이터로 분리했습니다.

```python
test_n = 62 * 24

train_df = df.iloc[:-test_n].copy()
test_df = df.iloc[-test_n:].copy()
```

또한 XGBoost 하이퍼파라미터 후보 비교 단계에서는 최종 test 데이터를 직접 튜닝에 사용하지 않기 위해, train 데이터 내부에서 마지막 1년치인 2024년 7~8월을 validation 데이터로 분리했습니다. 이후 선택된 모델을 다시 전체 train 데이터로 재학습하고, 최종 test 데이터에서 성능을 확인했습니다.

## 5. 주요 변수

| 변수명 | 설명 |
|---|---|
| `power_demand_MWh` | 제주도 시간별 전력 수요량(MWh), 종속 변수 |
| `is_weekend` | 주말 여부 |
| `tropical_night` | 열대야 여부 |
| `hour_sin`, `hour_cos` | 시간대의 순환적 특성을 반영한 변수 |
| `resident_pop` | 거주 인구 |
| `working_pop` | 근무 인구 |
| `visitor_pop` | 방문 인구 |
| `total_visitors` | 전체 관광객 수 |
| `renewable_energy_mwh` | 재생에너지 발전량(MWh) |
| `TA_mean` | 평균 기온 |
| `HM_mean`, `HM_max` | 평균 습도, 최고 습도 |
| `CDD` | 냉방도일 |
| `power_1h` | 1시간 전 전력 수요 |
| `power_48h` | 48시간 전 전력 수요 |

초기 후보 변수에는 `power_24h`, `power_168h`도 포함했지만, VIF 확인 결과 다중공선성이 높아 최종 모델에서는 제외했습니다. 단, `power_1h`는 전력 수요의 시간적 자기상관을 반영하는 핵심 변수이므로 유지했습니다.

## 6. 분석 과정

### 6.1 데이터 전처리

- 시간 순서 기준 데이터 정렬
- 결측치 확인
- 시간 변수 생성 및 순환 변환
  - `hour_sin`
  - `hour_cos`
- 주말 여부, 열대야 여부, 냉방도일 변수 생성
- 과거 전력 수요 파생변수 생성
  - `power_1h`
  - `power_24h`

- 수치형 변수 표준화
- VIF 기반 다중공선성 점검 및 변수 제거

### 6.2 모델링

본 프로젝트에서는 Linear Regression과 XGBoost Regressor를 사용했습니다.

#### Linear Regression

선형 회귀 모델은 각 독립변수가 전력 수요에 미치는 방향성과 영향력을 해석하기 위해 사용했습니다. 회귀계수와 OLS 결과를 함께 확인하여 주요 변수가 통계적으로 유의한지 검토했습니다.

#### XGBoost Regressor

XGBoost는 비선형 관계와 변수 간 상호작용을 반영하기 위해 사용했습니다. 최종 모델은 과적합을 줄이기 위해 비교적 얕은 트리와 낮은 학습률을 사용했습니다.

최종 XGBoost 주요 하이퍼파라미터는 다음과 같습니다.

| 파라미터 | 값 |
|---|---:|
| `n_estimators` | 300 |
| `learning_rate` | 0.05 |
| `max_depth` | 3 |
| `min_child_weight` | 3 |
| `subsample` | 0.8 |
| `colsample_bytree` | 0.8 |
| `reg_lambda` | 1.0 |

하이퍼파라미터 후보 비교에서는 train 데이터 내부에서 validation 구간을 따로 분리하여 RMSE와 train-validation 성능 차이를 함께 확인했습니다. 트리 깊이를 4로 늘린 모델은 validation 성능은 더 좋았지만 최종 test 성능 개선으로 이어지지 않아, 더 안정적인 현재 모델을 최종 선택했습니다.

## 7. 모델 성능

테스트 데이터 기준 성능은 다음과 같습니다.

| 모델 | R² | MSE | RMSE |
|---|---:|---:|---:|
| 단순 모델(`power_1h`만 사용) | 0.9191 | 1630.5859 | 40.3805 |
| Linear Regression(최종 15개 변수) | 0.9731 | 542.1959 | 23.2851 |
| XGBoost Regressor | 0.9946 | 109.0331 | 10.4419 |

단순히 1시간 전 전력 수요만 사용하는 모델도 높은 설명력을 보였지만, 기상·시간·인구·관광객·재생에너지 변수 등을 함께 반영한 다변량 모델에서 성능이 크게 개선되었습니다. 최종적으로 XGBoost Regressor가 가장 낮은 RMSE와 가장 높은 R²를 기록했습니다.

## 8. 주요 인사이트

### 8.1 과거 전력 수요의 영향이 가장 큼

XGBoost 변수 중요도에서 `power_1h`가 가장 높은 중요도를 보였습니다. 이는 전력 수요가 직전 시간대의 수요와 강한 연속성을 가지며, 단기 예측에서 과거 전력 수요 변수가 매우 중요한 설명력을 가진다는 것을 의미합니다.

### 8.2 기온은 여름철 전력 수요 증가와 밀접하게 연결됨

`TA_mean`은 XGBoost 모델에서 높은 중요도를 보인 변수 중 하나입니다. 여름철 평균 기온 상승은 냉방 수요 증가로 이어지며, 전력 수요 예측에서 중요한 기상 요인으로 작용했습니다.

### 8.3 시간대별 수요 패턴이 존재함

`hour_sin`, `hour_cos` 변수를 통해 시간대의 순환적 특성을 반영했습니다. 전력 수요는 하루 중 시간대에 따라 반복적인 패턴을 보이므로, 단순한 정수형 hour보다 순환 변환을 적용한 변수가 모델링에 적합했습니다.

### 8.4 다변량 모델이 단순 모델보다 우수함

`power_1h` 단일 변수 모델도 높은 성능을 보였지만, 최종 다변량 선형 회귀 모델의 R²가 더 높고 RMSE가 낮았습니다. 이는 전력 수요가 단순히 직전 수요만으로 결정되는 것이 아니라 기상, 시간, 인구, 관광객, 재생에너지 등 다양한 요인의 영향을 받는다는 점을 보여줍니다.
